# F4 商品搜索/筛选/分页体验

你现在作为前端交互与性能优化专家，帮助我实现 Backlog F4「搜索/筛选/分页」对应的前端功能。

上下文文档：
- frontend/plan-backlog-frontend.md 中 F4 条目
- frontend/前端开发计划.md 中「搜索体验」和性能相关部分
- backend/api-spec.md 第 2.1「搜索商品」

前置假设：
- F2 商品列表/详情已实现基础列表和详情展示；
- 已有 /items 路由与列表页面。

请在前端实现和优化：
1. 查询参数与表单：
   - 在 `/items` 页面中增加：关键词输入框、分类选择、价格区间（minPrice/maxPrice）等筛选控件；
   - 使用 URL 查询参数同步当前筛选条件（例如 `?keywords=xx&category=yy&minPrice=..&maxPrice=..&page=1`）。
2. 分页与列表：
   - 使用后端分页（page/size），设计分页组件（Pagination），支持下一页/上一页和页码跳转；
   - 控制 page size 上限，防止一次请求过多数据；
   - 在加载过程中使用 Skeleton 或 Loading 状态，在无结果时展示空态说明。
3. 状态管理：
   - 将当前筛选条件与分页信息纳入状态管理（如 Zustand store 或 URL 驱动的局部状态）；
   - 设计缓存策略：同一筛选条件下翻页的体验与缓存失效策略。
4. 性能与用户体验：
   - 避免每次输入字符都立即触发请求（使用防抖/提交按钮策略）；
   - 对于返回列表较长的情况，可讨论是否需要虚拟列表（可选，给出建议）；
   - 确保满足前端开发计划中的性能目标（P95≤2s）至少从交互设计上不制造额外瓶颈。

请输出：
- `/items` 页面中搜索与筛选部分的组件设计（SearchBar/FilterPanel），及与 URL 同步的示例代码；
- 分页组件与请求封装示例；
- 状态管理与缓存 key 设计说明；
- 针对防抖/节流与用户体验的文字建议。
