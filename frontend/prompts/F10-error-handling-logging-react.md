# F10 前端错误码与日志统一

你现在作为前端错误处理与可观测性专家，帮助我实现 Backlog F10「错误码与日志统一」对应的前端工作。

上下文文档：
- frontend/plan-backlog-frontend.md 中 F10 条目
- frontend/前端开发计划.md 中「错误、日志与可观测」章节
- backend/api-spec.md 第 0 章约定与第 6 章错误码

前置假设：
- 主要业务页面（F1–F8）已有基础实现；
- 已有 Axios 或统一的请求封装。

请在前端实现：
1. 错误码到文案映射表：
   - 设计一个集中式的错误码 → 用户可读文案映射（如 AUTH_REQUIRED、AUTH_FAILED、NOT_FOUND、INVALID_AMOUNT 等）；
   - 支持根据接口上下文附加额外说明（如在哪个页面发生）。
2. Axios 拦截器或请求封装：
   - 在响应拦截器中统一处理后端的错误格式 `{code, message}`；
   - 根据错误码进行分类处理（例如：401 → 跳转登录、403 → 权限提示、404 → 显示 NotFound 页面、其他 → Toast）。
3. 全局错误展示：
   - 设计一个全局错误提示机制（如 Toast/Modal/ErrorBanner）；
   - 对于致命错误，提供错误边界组件，展示兜底页面并提示用户刷新或重试。
4. 前端日志与简单埋点（可选）：
   - 设计一个简单的日志工具，将关键错误和操作打印到控制台或发送到后端日志接口（如有）；
   - 在认证失败、发布失败、报价失败等关键路径记录必要信息（不包含敏感数据）。

请输出：
- 错误码映射配置示例（如一个常量对象或函数）；
- Axios 拦截器/请求封装的示例代码；
- 全局错误提示与错误边界组件的设计说明。
