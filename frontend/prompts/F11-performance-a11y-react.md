# F11 性能与可访问性硬化

你现在作为前端性能与可访问性（A11y）专家，帮助我实现 Backlog F11「性能与A11y硬化」对应的工作，为整体前端做最后一轮打磨。

上下文文档：
- frontend/plan-backlog-frontend.md 中 F11 条目
- frontend/前端开发计划.md 中「性能与可访问性」章节

前置假设：
- F0–F10 的主要功能已完成并可运行。

请在前端从以下几个方面进行优化与检查：
1. 性能优化：
   - 分析主要路由的代码分割和懒加载（如详情页、发布页、管理页）；
   - 对列表与图片做懒加载或虚拟化的建议；
   - 检查并优化关键路径上的不必要重渲染（如使用 React.memo、合适的依赖数组等）。
2. 加载体验：
   - 为主要页面补齐 Skeleton/占位；
   - 确保分页和筛选时体验流畅，无明显布局抖动；
   - 对慢接口给出超时和重试提示策略。
3. 可访问性：
   - 检查并补充语义化标签（button/link/heading/form/label 等）；
   - 确保表单控件均有 label 和错误提示，并可通过键盘操作；
   - 为弹窗/对话框、Toast 等提供合适的 ARIA 属性与焦点管理（例如打开对话框时聚焦、关闭时返回触发点）。
4. 检查清单与自动化：
   - 设计一个简单的 A11y 检查清单（可人工检查 + 自动化工具如 Lighthouse/axe）；
   - 为关键页面添加 1~2 个自动化检查示例（如使用 Playwright/Cypress 调用 Lighthouse 或 axe）。

请输出：
- 针对现有路由和组件的性能/A11y 问题检查框架（可以是 Todo 清单形式）；
- 若干优化建议与代码示例（如如何处理长列表、如何改造当前弹窗实现以满足 A11y 基线）；
- 建议引入的自动化工具与配置示例。
