前置假设：
- 已有 `/me/posts` 路由页面占位；
- 认证与基础发布功能（F1/F3/F5）已完成。

请在前端实现：
1. 我的发布列表页 `/me/posts`：
   - 展示当前用户发布的商品和需求列表（可以区分 Tab：商品/需求）；
   - 调用相应 API 获取当前用户资源（若后端未提供专门接口，可先通过前端过滤或补充设计说明）。
2. 更新与下架/删除操作：
   - 对每一条资源提供操作入口（如「编辑」「下架」「删除」按钮）；
   - 触发后调用相应 PATCH 接口（Items/Demands），传入 `{action: "update|off"}` 或使用后续软删接口；
   - 对成功/失败进行 Toast 提示；

3. 权限与错误反馈：
   - 处理 403 `FORBIDDEN_OWNER`，在 UI 上提示用户无权操作该资源；
   - 处理 409 `CONFLICT_STATE`，提示当前状态不允许此操作（如已 deleted）。
4. 编辑表单（可选）：
   - 复用发布商品/需求的表单作为编辑表单，预填现有数据；
   - 编辑成功后刷新详情与我的发布列表。
