# F9 审核台（可选）前端实现

你现在作为前端管理后台与权限控制专家，帮助我实现 Backlog F9「审核台（可选）」对应的前端功能。

上下文文档：
- frontend/plan-backlog-frontend.md 中 F9 条目
- frontend/前端开发计划.md 中「运营审核」阶段说明
- backend 侧关于 review.enabled 开关与审核流程的约定（见开发启动包和 API 设计）

前置假设：
- 审核开关 `review.enabled` 在后端已实现；
- 已有 `/admin/review` 路由页面占位；
- 已有基础认证与角色管理（如 user/admin）。

请在前端实现：
1. 审核列表页 `/admin/review`：
   - 展示待审核的商品/需求列表（pending 状态）；
   - 字段包含标题、类型（商品/需求）、提交人、提交时间等。
2. 审核操作：
   - 每条记录提供「通过」「拒绝」操作按钮；
   - 调用后端相应审核接口（如果尚未定义，给出前端期望的接口契约示例，如 PATCH /api/items/{id} action=reviewApprove/reviewReject 等）；
   - 操作成功后，从列表中移除该条记录，或更新为已审核状态。
3. 权限控制：
   - 仅管理员角色可访问 `/admin/review`；
   - 若非管理员访问，重定向或展示 403 页面；
   - 根据后端返回 403 错误码进行 UI 提示。
4. 体验与可观测性：
   - 批量审核能力（可选，给出设计建议即可）；
   - 审核操作结果的日志/埋点建议（如记录操作人和目标资源）。

请输出：
- `/admin/review` 页面结构设计与主要组件（如 ReviewTable/ReviewFilters）；
- 与后端审核接口的期望契约和调用示例；
- 权限守卫与路由控制示例代码。
